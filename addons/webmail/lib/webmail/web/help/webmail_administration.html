WebMail administration

WebMail administration performs just through the single Administration Console.

Navigate to "http://path_to_webmail/mailadm.php" page in your browser. Use "mailadm" as Login name.
Type your default admin password "12345". It is strongly recommended to change this password.

After login Administration Console appears with the following menu:

 

Administration Console:

Value Description
Database settings Use this section to set the database type you want to use by WebMail.
Users management Use this section to manage WebMail users.
WebMail settings Use this section to set settings of WebMail system, such as Site Name, POP3/IMAP4/SMTP Servers and other default settings.
Interface settings Customize WebMail's look and feel: skin, default language, etc.
Login settings Customize login window look and functionality.
Debug settings Logging mail sessions into a file, etc.
Mail Server Integration Mail Server Integration settings, etc.

 

Folowing items of top menu are available:
Value Description
Return to mail login form Exits WebMail administration panel, and opens user login page.
Help Opens WebMail documentation.
Logout Exits WebMail administration panel, and returns to the administration login page.

 

Database Settings Section.

Selects what type of database WebMail should use, MS SQL Server or MySQL Server:

MS SQL Server / MySQL Server

Value Description
SQL login Specifies Login name to access SQL database.
SQL password Specifies Password to access SQL database.
Database name The name of database in SQL server where you want WebMail to operate with.
All required tables are created automatically ones by WebMail, when you first time enter database name.
Data source (DSN) Specify Data Source Name if you wish to connect through DSN to database.
Host Specify Host field if you wish to connect through TCP/IP

 

Users Management section

Enables to view complete list of registered users in WebMail system database.
Administrator can create new user accounts, edit profiles and delete selected users from database. Search feature is also available.

 

WebMail Settings section.

Here you can set some parameters and default values.

Value Description
Site name Title that will be shown in browser's header.
Incoming mail Incoming mail server name or IP, such as "pop.server.com".
Incoming mail port Incoming mail server port (by default, 110 for POP3 and 143 for IMAP4).
Outgoing mail Outgoing mail server name or IP, such as "smtp.server.com".
Outgoing mail Port Outgoing mail server port (25 by default).
Requires SMTP authentication If checked, SMTP authentication will be enabled by default.
Allow direct mode This option allows or forbids users to set Direct Mode for their POP3 accounts and IMAP folders.
Direct mode is default This option makes all new POP3 accounts and all IMAP folders be synchronized in Direct Mode by default. In Direct Mode, MailBee WebMail Pro works directly with the mailbox on mail server (mails are always taken directly from mail server), database is used only to store user accounts and address books (mails are not stored in database).
Attachment size limit Specifies maximum allowed size (in bytes) of the file to be attached to the message during composing. If the attachment file size exceeds specified limit, it will be rejected (Note that your web server may also imply certain restirictions on maximum size of the data being uploaded. See FAQ for details).
Enable mailbox size limit Specifies whatever to limit mailbox size or not. Set it to the value of user's maximum mailbox size (in bytes).
Allow users to change email settings This option allows or forbids users to change their account settings ("Friendly Name" can be edited anyway).
Allow users to add new email accounts This option allows or forbids creating multiple email accounts per user.
Allow automatic registration of new users on first login This option allows or prohibits automatic user registration in WebMail Pro on first login. If this option is turned off, new users can be added to WebMail Pro only by administrator via "WebMail Administration Console".
Default user charset and Allow users to change charset Default charset for assigned to messages composed by the user, and permission to change this charset by users themselves.
Default user time offset and allow users change time offset Default Timezone Offset, and permission to change it by users themselves.
New administration password Specifies WebMail Administrator password. Sets to default value "12345" after WebMail installation. It is strongly recommended to change it after installation. If these fields are left blank, existing password remains unchanged.
Confirm password

 

Interface Settings section.

Value Description
Mails per page Mail list is divided into pages. Each page holds specified number of mail messages to display.
Default skin Specifies default skin for WebMail appearance.
Allow users to change skin Specifies if users are allowed to change skin of WebMail appearance.
Default language Default interface language.
Allow users to change interface language Permission to change interface language by users.
Show text labels If checked, text labels will be displayed for the icons on toolbars.
Allow AJAX Version If checked, AJAX interface will be enabled.
Allow DHTML editor If checked, allows users to compose messages in HTML mode.

 

Login Settings section.

Value Description
Standard Login Panel All fields are displayed (email, login, password).
Hide login field Hide login field and take it from email:
  1. Use Email as Login - whole email address is used as login (e.g. if email is jdoe@domain.com, login will be jdoe@domain.com)
  2. Use Account Name as Login - truncate email address from "@" and use the remainder (e.g. if email is jdoe@domain.com, login will be jdoe)
Hide email field Hide Email field and build e-mail address from login field and, optionally, from domain
  1. Display Domain After Login Field - on login form, "@"domain will be displayed after login field. This parameter affects only visual look of login panel. Decision on how to build login or email address is not altered.
  2. If ticked, WebMail will construct email account login as concatenation of Login field contents + "@" + domain. You should use this if logins on your mail server look like jdoe@domain.com (i.e. they are the same as email address). Do NOT use this option if logins look like jdoe (e.g. just account name, without domain).
Allow advanced login If checked, clients of the WebMail system will be able to use the system with POP3/IMAP4/SMTP servers different from specified in POP3/IMAP4/SMTP Server settings.
Automatically detect and correct if user inputs e-mail instead of account-name. Allow WebMail to detect and fix when user inputs email instead of login, or login instead of email, and it's possible to determine correct login or e-mail from typed. Example: if user inputs full email address in Login field (while only account name is expected), WebMail will automatically drop @domain part.

 

Debug Settings section.

Value Description
Enable Logging Enable logging of mail operations (POP3/IMAP4/SMTP) into a file. Path For Log box specifies physical location (e.g. C:\Data\log.txt for Windows or /pub/webmail/data/log.txt for Unix) where to create the log.
Clear log Clear log file.
Show All Log Show all log records.
Show Log (last records) Show last 50K of log records.

 

Mail Server Integration section.

Value Description
Enable Integration Allows managing accounts on AfterLogic XMail Server from MailBee WebMail Pro.
Path to Server Path to the MailRoot folder of AfterLogic XMail Server in your system, for instance C:\Program Files\XMail\MailRoot\.
