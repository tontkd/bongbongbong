AfterLogic XMail Integration Instructions for (Unix) Windows

  1. Download and install AfterLogic XMail Server using the installation instructions shipped with the product.


  2. Make sure you've created at least one domain in AfterLogic XMail Server configuration. This step is required for proper integration of the products.


  3. Grant full control permission to web server process:
    - apache.apache user for Apache
    - Internet Guest Account (IUSR_YOUR_SERVER_NAME) user for IIS
    over XMail/MailRoot/domains folder and wm.tab file.

  4. In WebMail Admin Console / Mail Server Integration, turn "Enable Integration" option on and specify correct path to XMail/MailRoot folder in "Path to Server" field. Also, you need to specify address of the server AfterLogic XMail Server is installed on in "Server Host" field. If you need accounts created/removed in WebMail to be created/removed on AfterLogic XMail Server too, enable "Allow users to manage accounts on AfterLogic XMail Server".

    In this case, if a user adds or removes a linked account in his primary account settings and domain part of this account matches any of your domains hosted by AfterLogic XMail Server, this account will be added/removed on AfterLogic XMail Server.
